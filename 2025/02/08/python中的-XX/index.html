1.3 __del__(self)

  • 适用场景：在对象生命周期结束时（例如关闭文件、释放网络资源），自动执行清理操作。

  • 注意事项：

    • 调用时机不确定：__del__ 方法的调用时机由 Python 的垃圾回收器决定，而不是在对象被显式销毁时立即调用。这可能导致 __del__ 方法的执行时间不可预测。

    • 依赖关系问题：如果对象之间存在复杂的依赖关系，__del__ 方法可能会导致不可预测的行为。例如，一个对象的 __del__ 方法可能会引用另一个已经销毁的对象，从而引发错误。

    • 与垃圾回收器的交互：在某些情况下，__del__ 方法可能会干扰垃圾回收器的正常工作，导致内存泄漏或其他问题。

  • 补充：Python 的垃圾回收机制

    Python 使用引用计数和周期性垃圾回收（GC）来管理内存：

    • 引用计数：每个对象都有一个引用计数器，记录指向该对象的引用数量。当引用计数降为零时，对象会被自动销毁。
    • 垃圾回收：Python 的垃圾回收器会周期性地检查并回收不可达对象（即没有引用的对象）。

    可以不进行空间回收。具体情况建议参考竞赛具体要求。

7.3 __copy__(self) 与 __deepcopy__(self, memo)

  • 适用场景：用于自定义对象的浅拷贝和深拷贝行为。当对象内部包含复杂可变数据结构，需要控制复制方式时。

  • 补充：浅拷贝与深拷贝

    浅拷贝（Shallow Copy）

    • 浅拷贝会创建一个新的对象，但其中的元素是原始对象的引用。这意味着，如果原始对象中的元素是可变对象（如列表、字典等），修改新对象中的这些元素也会影响原始对象。
    • 实现方式：
      • 使用 copy.copy() 方法。
      • 使用切片操作符 [:]。
    • 特点：
      • 浅拷贝只拷贝父对象，不拷贝子对象。
      • 对于嵌套数据结构，浅拷贝后，嵌套的子对象仍然是原始对象的引用。

    深拷贝（Deep Copy）

    • 深拷贝会创建一个新的对象，并且递归地拷贝原始对象中的所有子对象。这意味着，原始对象和新对象是完全独立的，修改一个不会影响另一个。
    • 实现方式：
      • 使用 copy.deepcopy() 方法。
    • 特点：
      • 深拷贝会拷贝对象及其所有子对象，确保新对象与原始对象完全独立。
      • 对于嵌套数据结构，深拷贝后，所有嵌套层级都是独立的。
